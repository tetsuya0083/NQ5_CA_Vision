3. Breaking Changes 상세 설명
--------------------------------------------------------------------------------

  3.1 BinaryFormatter 제거
  ────────────────────────
  .NET 8에서 BinaryFormatter는 완전히 비활성화되었습니다.
  Common\Util.cs의 object2ByteArray, ByteArray2object 등 메서드에서 사용했으나,
  이미 주석 처리되어 있고 return null을 반환하는 상태였습니다.
  using 선언만 삭제하여 처리했습니다.
  ※ VisionSetup의 ModelSetupForm.resx에 ImageList의 ImageStream이 BinaryFormatter로
     직렬화되어 있어 경고가 발생합니다. .NET 8에서는 동작하지만 .NET 9부터 문제가
     될 수 있으므로 추후 리소스 파일 재생성을 권장합니다.

  3.2 System.Data.SqlClient → Microsoft.Data.SqlClient
  ─────────────────────────────────────────────────────
  .NET 8에서는 System.Data.SqlClient가 기본 포함되지 않습니다.
  Microsoft.Data.SqlClient NuGet 패키지(5.2.0)로 대체했습니다.
  API는 완전 호환(SqlConnection, SqlCommand, SqlDataReader 등 동일)이지만,
  기본 동작이 다른 점이 있습니다:
    - Encrypt 기본값: false → true (변경됨)
    - 내부 네트워크 SQL Server에서 TLS 인증서 없이 연결하면 실패할 수 있음
    - 대응: 연결 문자열에 Encrypt=false;TrustServerCertificate=true 추가

  3.3 Encoding.Default 동작 변경
  ──────────────────────────────
  .NET Framework: Encoding.Default = 시스템 ANSI 코드페이지 (한국어 Windows → EUC-KR/CP949)
  .NET 8:         Encoding.Default = UTF-8
  로그 파일, 카메라 데이터 등에서 사용되고 있어 명시적으로 Encoding.UTF8로 변경했습니다.
  영향받는 위치 4곳:
    - Common\Util.cs (로그 쓰기, RemoveNull)
    - Common\Data.cs (파일 쓰기)
    - Common\CameraObject.cs (카메라 데이터 수신)

  3.4 COM Interop (XGCommLib)
  ───────────────────────────
  dotnet build(MSBuild .NET Core 버전)에서는 COMReference의 ResolveComReference
  작업을 지원하지 않습니다 (MSB4803 오류).
  해결 방법: 기존 빌드에서 생성된 Interop.XGCommLib.dll을 lib\ 폴더에 복사하고,
  csproj에서 직접 Reference로 참조하도록 변경했습니다.
  EmbedInteropTypes=true를 유지하여 런타임에 별도 DLL이 필요하지 않습니다.

  3.5 SDK-style csproj 변환
  ─────────────────────────
  old-style csproj → SDK-style csproj로 전면 교체했습니다.
  주요 차이점:
    - .cs 파일 자동 포함 (개별 Compile Include 불필요, 단 Link 파일은 명시 필요)
    - packages.config 제거 → PackageReference 사용
    - AssemblyInfo.cs 자동 생성 (수동 파일 삭제)
    - Framework Reference 자동 포함 (System, System.Core 등 명시 불필요)
    - UseWindowsForms=true로 WinForms 및 System.Drawing 자동 참조
